# Quickstart

Run the PRISM-IR round-trip on a worked example in under two minutes.

## Prerequisites

- Python 3.11 or newer
- An Anthropic API key (any plan)

## Steps

```bash
# 1. Clone the repo
git clone https://github.com/deiasolutions/prism-ir.git
cd prism-ir

# 2. Install dependencies
pip install anthropic pyyaml

# 3. Set your API key
export ANTHROPIC_API_KEY=sk-ant-...

# 4. Run the round-trip
python scripts/roundtrip.py examples/claims-processing.prism.md
```

## What you should see

The script prints two blocks side by side: the original English process
description from the file, and a reconstruction generated by an LLM
that has only seen the IR.

The original looks like this:

```
1. Customer submits claim
2. System validates policy status
3. If policy inactive, reject claim
4. If claim amount exceeds threshold, route to manual review
5. Otherwise approve automatically
6. Notify customer and archive decision
```

The reconstruction will be different in surface wording but should
preserve every step, every branch, every consequence. If it does, the
IR is faithful — the round-trip principle holds.

## Try the other examples

```bash
python scripts/roundtrip.py examples/expense-approval.prism.md
python scripts/roundtrip.py examples/sir-epidemic.prism.md
python scripts/roundtrip.py examples/l-system-tree.prism.md
```

These cover three different domains — finance, epidemiology, generative
grammar — running through the same IR machinery.

## What this is and isn't

This is a demonstration. The round-trip output is human-readable, and
you compare the two descriptions yourself. Phase 3 of the build adds a
verifier that uses an LLM-as-judge call to score semantic equivalence
automatically and gates CI on the result. For now: clone, run, read.
